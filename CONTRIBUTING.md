# Contributing to TYRP

Thanks for helping improve TYRP.

## Before you start

- Check existing issues and pull requests to avoid duplicate work.
- Keep changes small and focused.
- Add tests when you change behavior.

## Development flow

1. Create a branch for your work.
2. Make the change.
3. Run `dart analyze` and `dart test`.
4. Open a pull request using the template.

## Pull requests

- Explain the problem and the solution.
- Include screenshots or terminal output when they help clarify the change.
- Call out anything that still needs follow-up.
